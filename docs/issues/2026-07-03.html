Codex 官方动态 & 使用技巧
🔒 wechat-article-publish 安全机制：草稿先行，发布必须确认
发布 Skill 的默认行为是只创建草稿——不会擅自发布。正式发布需要你明确说「确认发布」。这个设计是因为公众号文章一旦发布就不可撤回（只能删除），AI 不应该替你做这个决定。使用方式：先草稿 → 微信后台预览 → 确认无误 → 再让 Codex 执行发布。
⚡ 技巧：微信凭据安全存放
不要把 WECHAT_APP_ID 和 WECHAT_APP_SECRET 硬编码在代码里。推荐放在 ~/.config/wechat-mp/.env 文件中——wechat-article-publish 会自动读取。同时确保在微信后台「基本配置」页面添加你的服务器 IP 到白名单，否则 API 调用会返回 IP 未授权错误。
📊 md2wechat 支持多账号矩阵号发布
md2wechat config wechat-accounts 可以查看本地多公众号账号配置。每个账号命名管理，本地只读发现，不输出 Secret。适合运营多个公众号的矩阵号场景——一条命令指定账号名，自动切换到对应凭据发布。

NotebookLM 每日进步
🧪 NotebookLM — 快速研究 vs 深度研究：怎么选
NotebookLM 现在有两种研究模式：快速研究适合 10 分钟搞清一个概念，深度研究适合 30 分钟出一份带引用的报告。公众号选题阶段用快速研究确认方向，写作阶段用深度研究补充数据——两个模式分阶段用，效率最高。
🧪 NotebookLM — 音频播客 + 笔记联动
生成 AI 播客后，播客里提到的每个观点都会自动关联到原始笔记中的对应段落。听播客时听到有意思的点，点击就能跳回原文查看详细数据。做选题调研时，先听播客快速过一遍，再点进去精读重点段落。
🧪 NotebookLM — 项目文档喂成个人研究助手
把你的公众号选题灵感库、过往文章、竞品分析报告全部上传到一个 NotebookLM notebook。之后每次有新选题，问「这个选题我之前写过类似的吗？」「同行有没有写过类似角度？」——它直接翻你的历史资料回答，比你自己翻文件夹快 10 倍。

WorkBuddy 实用技巧
⚡ 分步指令+检查点：发布流程的安全带
发布是高风险操作——用分步指令+检查点确保安全：「第1步：用 $wechat-article-publish 创建草稿 → 完成后暂停让我确认预览 → 第2步：确认无误后正式发布」。WorkBuddy 会在每步完成后暂停等你确认，不会一口气发出去。
⚡ Ask 模式先收集，再 Craft 出结果
调研类任务先用 Ask 模式确认方向（只读不写、消耗低），确认信息够用了再切 Craft 模式出报告/文章。中间来回追问不会白白消耗积分。发布前的选题确认、配图确认，都用 Ask 模式，最后发布用 Craft 模式。
