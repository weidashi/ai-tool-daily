ChatGPT Work 来了！Codex 并入桌面端，一个 App 搞定聊天、开发和交付

📌 本期导读
7 月 9 日当天，GPT-5.6 之外还有三个重磅发布：ChatGPT Work 智能体、Codex 并入 ChatGPT 桌面端、ChatGPT Sites 公开 Beta。这期我们跳出 Skills，看看平台层面的变化如何改变你的日常工作方式。前四天装的 Skills 在 Work 中全部可用。
🆕
三个平台级更新
①
ChatGPT Work — 不只是聊天，是真正的"干活"
跨应用搜集信息 → 拆解复杂任务 → 产出可交付成果
重磅GA
是什么：Work 是一个可以连续工作数小时的智能体。它不只回答问题——它能跨你的文件、应用、工作流搜集信息，产出文档、表格、PPT、网页应用等"完整交付物"。你可以中途看进度、回答问题、改变方向、批准关键操作。

五个关键能力：
1. 自主拆解任务——把一个目标（"做一份市场分析报告"）自动拆成多步骤
2. 跨应用工作——读取你的文件、连接的应用，在不同工具间流转信息
3. Scheduled Tasks——任务可以定时、重复、条件触发或监控变化
4. 桌面端能力——在 macOS/Windows 桌面端可访问本地文件和桌面应用（需授权）
5. 内置浏览器——收集网页信息、操作支持的 web 工具和文件

最适合：需要多个步骤、跨多个工具才能完成的任务——以前找 3 个 AI 轮番干的事，现在 Work 一个搞定。
②
Codex 并入 ChatGPT 桌面端
Chat + Work + Codex 三合一，不再需要三个 App
重磅合并
是什么：全新的 ChatGPT 桌面端把 Chat（问答）、Work（交付）和 Codex（开发）整合到一个 App 中。Codex 作为独立视图保留，同时新增了行内编辑 diff、GitHub PR 侧面板审核、多仓库项目管理、更快的 Computer Use 等专属功能。

关键变化：
• Codex 可设为默认视图——打开就是开发界面
• Work 可调用 Codex 能力——非开发者也能用代码生成网页、分析数据
• 桌面端文件访问——Codex 可直接编辑本地文件
• PR 审核侧面板——评审意见同步显示在 diff 旁边
• 多仓库项目——一个 Codex 项目可以跨多个 Git 仓库工作
③
ChatGPT Sites — 把想法变成可分享的网站
无需编程，描述需求 → 预览 → 发布
Beta新品
是什么：在 Work 或 Codex 中描述你想要的东西，ChatGPT 生成一个交互式网站或轻量应用——仪表盘、项目追踪器、产品日历、原型、内部门户。Business/Enterprise 用户可以公开发布并分享 URL。

和 Word/PPT 的区别：Sites 是"活的"——可以包含交互式图表、实时数据、用户输入。比文档更有表现力，比传统网站开发快 100 倍。
📘
三大场景实战：Work 怎么用
📊 场景一：从零散的会议记录到完整汇报
"把上周三次会议的记录、客户邮件和销售数据整理成一份 Q2 市场分析报告，包含趋势图、竞品对比表和 3 条建议。PPT 格式，商务蓝配色。" → Work 自动搜集材料、分析数据、生成文档。以前要一天，现在 30 分钟。
📱 场景二：一次输入，多平台分发
"把这份市场分析报告改写成公众号文章（1500字）、小红书文案（800字+8张卡片）、LinkedIn 英文帖（300词）。保持核心观点一致，适配各平台风格。" → Work 并行处理，5 分钟后拿到三个版本。
⏰ 场景三：定时监控 + 自动汇报
"每天早上 9 点，检查竞品A、B、C的官网和公众号更新，如果有新产品发布或定价变化，立即生成一份简报发给我。" → Scheduled Tasks 让 Work 在你睡觉时也在干活。

NotebookLM + Work 协同
🔗 研究→输出流水线
NotebookLM 做深度研究（上传资料→提问→交叉验证）→ 导出笔记 → 喂给 ChatGPT Work → Work 把笔记转成报告+PPT+网站。两个工具各司其职：NotebookLM 专注"理解"，Work 专注"产出"。
📙
WorkBuddy + Work 协同
🔄 本地自动化 + 云端编排
WorkBuddy 擅长时间触发的本地任务（数据采集、文件生成、Git 推送），Work 擅长复杂编排和多应用分发。两者组合：WorkBuddy 生成内容 → 推送到 GitHub → Work 读取并分发给各平台。一个管"制造"，一个管"物流"。

今日成果展示
🎯 从 0 到完整交付：用 Work 跑一个真实任务
以前需要 3 个工具、整个下午的事，现在一个 Work 搞定
任务："分析过去 3 个月我写的 12 篇文章，找出阅读量前 3 的共性（选题角度、结构特点、标题特征），生成一份优化建议报告，并基于建议写一篇新的公众号文章。"

ChatGPT Work 自动做了：
1. 读取 12 篇文章 → 提取标题、结构、阅读量
2. 分析共性 → 生成数据可视化图表
3. 写优化建议报告（PDF）
4. 基于建议写新文章草稿
5. 生成新文章的封面图
耗时：约 25 分钟。手动做至少 3 小时。
🌱 ChatGPT 不再是"聊天工具"，是能帮你干活、交付、甚至在你不在时值班的工作伙伴。
