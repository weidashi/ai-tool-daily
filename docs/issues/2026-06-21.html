今天全部是 非编程、纯日常 的 Codex Skills——写作润色、PPT 生成、选题诊断、内容创作、社交卡片封装。适合不做开发的知识工作者：写公众号、做汇报 PPT、产出深度文章、运营小红书。5 个 Skill 都复制即装，半小时内跑通一个完整创作流。
📘Codex 日常效率 Skills（5 个必装）
① Humanizer-zh 中文必备 去AI味 写作润色
是什么：给中文文章"去 AI 味"。能识别空话、套话、三段式、过度排比、AI 腔，把文章改得像真人写的。作者是 op7418（guizang-ppt-skill 同作者）。
最适合：任何需要写中文文章的人——公众号、小红书、报告、邮件、方案。写完后一键"去 AI 味"。
# 安装（Codex CLI） npx -y skills@latest add op7418/Humanizer-zh --skill humanizer-zh --agent codex --global # 或在 Codex 里直接说 "请帮我安装 Humanizer-zh skill，GitHub 链接是 https://github.com/op7418/Humanizer-zh"
🎯 今天就能试：写一段文章，然后对 Codex 说 "用 humanizer-zh 润色这段文字，去掉 AI 味，让它更像真人写的" → 自动识别并修复 AI 腔、套话、过度排比。

💡 用法：不是"帮我写一篇文章"，而是"我先写 → 再让它帮我改"。先用自己的想法写初稿，再用 Humanizer-zh 做最后一道"去味"——这样文章保留你的观点，只有表达被优化。
② codex-ppt-skill v0.5.2 昨日发布 PPT 生成 10种风格
是什么：把文章、报告、论文、笔记、Markdown 转成图片式 PPT。作者 ningzimu 把做 PPT 拆成 11 步可审阅的工作流：先确认大纲 → 选视觉风格 → 看 1 页样张 → 确认后批量生成 → 组装成 .pptx。不是"点一下出 20 页 PPT"，而是把 AI 做 PPT 变成一个像真实工作流的过程。
昨天（6月20日）刚发布 v0.5.2。
# 安装（一句话发给 Codex） "请帮我安装 codex-ppt skill，GitHub 链接是 https://github.com/ningzimu/codex-ppt-skill" # 或 CLI 安装 npx -y skills@latest add ningzimu/codex-ppt-skill --skill codex-ppt --agent codex --global
🎯 今天就能试：准备一篇你写的 Markdown 文章或笔记，对 Codex 说 "用 codex-ppt skill 把这份内容转成 PPT，风格选手绘技术解释风" → 先确认大纲 → 看样张 → 批量生成 10 种内置风格任选。
🖐️ 5 个 fork 场景：① 技术文章 → 分享 PPT ② 论文/报告 → 演示稿 ③ 课程笔记 → 课件 ④ 商业汇报 → 方案 PPT ⑤ 读书笔记 → 分享卡片。你只需要提供内容，PPT 格式和视觉风格由 AI 处理。
③ dbskill 自媒体必备 选题诊断 爆款拆解
是什么：内容创作者的"选题诊断器"。帮你诊断选题质量、优化商业表达、拆解爆款结构、生成小红书标题。
最适合：中文自媒体、知识型创作者。发文章前用 dbskill 过一遍选题，避免写了两小时发现选题本身没吸引力。
# 安装 npx -y skills@latest add dontbesilent/dbskill --skill dbskill --agent codex --global
🎯 今天就能试：想三个文章选题，对 Codex 说 "用 dbskill 帮我诊断这三个选题，哪个最有可能火？" → 从热点、受众、角度等维度给诊断报告。

💡 日常用法：写文章前问一句，发小红书前问一句——每次只需 2 分钟，但能让你少写很多没人看的内容。
④ content-research-writer 完整写作流 深度文章
是什么：从选题、资料收集、提纲、引用到初稿，一整条写作工作流直接跑通。来自 ComposioHQ 仓库。
最适合：写公众号长文、Newsletter、深度博客、行业报告——需要查资料、引用来源的严肃写作。
# 安装（ComposioHQ 仓库） git clone https://github.com/ComposioHQ/awesome-codex-skills.git cp -r awesome-codex-skills/content-research-writer ~/.codex/skills/content-research-writer
🎯 今天就能试：对 Codex 说 "用 content-research-writer 帮我写一篇关于'2026年 AI 编程工具格局'的深度分析文章，包含最新行业数据" → 自动搜资料 → 出提纲 → 写正文 → 标注引用来源。

💡 和 dbskill 的区别：dbskill 负责"这个选题行不行"，content-research-writer 负责"这篇怎么写"。先诊断再写，顺序不要反。

🧩 今天的"完整创作流"组合方案
把这 5 个串起来，就是一条完整的中文内容创作流水线：

dbskill（选题诊断，花 2 分钟确认值不值得写）→ content-research-writer（搜资料出提纲写初稿）→ Humanizer-zh（去 AI 味，让文章像真人写的）→ codex-ppt-skill（把核心观点转成分享 PPT）→ social-card-skill（拆成小红书卡片，多平台分发）

📌 今天装这 5 个，半小时内就能跑通一篇从选题到分发的完整文章。
🆕Codex 日常使用提升技巧
⚡ 技巧 1：用 Codex Appshots 一键让 AI 看懂你的屏幕
是什么：Codex macOS 版 5 月 21 日新功能——按两下 Command 键，直接把当前窗口的截图+文本发给 Codex。不用复制粘贴。

日常用法：看到一篇好文章、一个排版很棒的页面、一份设计参考？按两下 Command → 对 Codex 说"按这个风格给我写一篇/做个 PPT"→ AI 直接看懂你的参考，不用文字描述"就是那种感觉的"。
⚡ 技巧 2：让 Codex 帮你读网页 / 总结文章
日常操作：对 Codex 说 "帮我打开 https://xxx，用 3 句话总结这篇文章的核心观点" → Codex 自动用应用内浏览器打开页面，提取原文内容并总结。

进阶：可以同时喂多个链接，让 Codex 跨文章对比观点："对比这篇文章和这篇文章关于 XX 的观点，分别列出论据"。
⚡ 技巧 3：给 Codex 建立"个人知识库"（CLAUDE.md）
场景：每次写文章都要告诉 Codex"我是做 XX 领域的，我的写作风格是……我的受众是……"

今天就能做：在 ~/.codex/CLAUDE.md 里写下你的个人信息（和项目级 CLAUDE.md 不同，这个全局生效）。内容示例：
## 我的身份 我是一名 XX 领域的从业者，日常需要写公众号文章和方案文档。 我的受众人群是：XX。我的写作风格偏：XX。 ## 格式偏好 - 所有文章用中文，小标题加粗不用编号 - 每段不超过 3 行，多用短句 - 结尾必须有总结段落
以后每次开 Codex，它自动记住这些，不用重复说。
📗待学习：NotebookLM & Notion 每日进步
🧪 NotebookLM — 技巧 1：用"笔记本笔记"当 AI 的行为指南
操作：在任意 NotebookLM 笔记本中，点左侧"笔记"面板 → 新建一条笔记 → 写入你的偏好。例如："本笔记本用于 XX 项目的资料整理，所有回答请用中文，优先引用来源原文，按时间线组织信息"。
之后每次在这个笔记本里提问，AI 都会自动参考这条笔记作为行为准则。
🧪 NotebookLM — 技巧 2：限制来源范围，聚焦分析
操作：在对话输入框上方点"来源"按钮 → 勾选你希望 AI 参考的具体文档 → AI 的回答只会基于这些文档。
笔记本里有几十份资料、但你只想分析其中一份时，这功能防止 AI 在无关文档里"翻箱倒柜"，回答更聚焦。
🧪 NotebookLM — 技巧 3：跨文档追问
操作：上传 3-4 篇同一主题的文档 → 在对话里问 "这三篇文档关于 XX 的观点分别是什么？哪篇论证最充分？" → AI 自动提取各文档的相关段落，并列对比。
适合做文献综述、行业研究时快速摸清不同来源的立场差异。
📋 Notion — 今日技巧：用按钮（Button）一键生成周报模板
操作：创建一个周报数据库 → 在页面输入 /button 插入按钮 → 命名为"新建本周周报" → 配置动作：在数据库添加新页面，预设"日期=今天""状态=草稿"→ 保存。
以后每周五点一下，周报页面自动创建，你再填内容即可。省掉每次手动新建+填字段的 3 分钟。
