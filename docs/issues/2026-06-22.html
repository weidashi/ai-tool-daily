📗 待学习：NotebookLM & Notion 每日进步
🧪 NotebookLM — 技巧 1：「选择来源」追查观点出处
操作：上传 3-4 篇同一话题的文档 → 在对话输入框点"来源"→ 勾选其中一篇 → 问"这篇文章的核心观点是什么？"→ 再切换勾选另一篇，问同样的问题 → 对比两篇回答差异。
适合做研究时逐篇精读 + 跨文档对比，不会让 AI 混淆来源。
🧪 NotebookLM — 技巧 2：用「朗读」模式消化长文
操作：在 NotebookLM 打开任意笔记本 → 点击右上角三个点菜单 → 选择"朗读"→ 系统朗读所有文档和笔记。通勤/做饭时听，把阅读时间"折叠"进碎片时间。 适合：需要看完但没时间坐下读的行业报告、政策文件、长论文。
🧪 NotebookLM — 技巧 3：用「笔记本描述」定制 AI 回答风格
操作：打开笔记本 → 左侧笔记面板 → 新建笔记 → 写入："请用简明中文回答，3 句话内给出结论，然后展开。回答时标注出处（文档名+页码）。优先使用原文引用而非改写。"
之后这个笔记本里所有 AI 回答自动遵守这段行为准则。
📋 Notion — 用按钮批量更新数据库字段
操作：在数据库页面输入 /button → 配置"编辑数据库页面属性"→ 设置多个字段同时更新（如「状态=完成」「完成日期=今天」）。选中多个条目后一键批量更新。
📋 Notion — AI 生成数据库模板
操作：新建数据库 → 点"使用 AI 生成"→ 描述需求（"一个内容创作项目管理数据库，包含状态、优先级、发布平台、字数"）→ AI 自动建好字段。
📙 实用技巧：WorkBuddy 与 Agent
⚡ 把 Codex Skills 同步到 WorkBuddy 用
场景：Codex Skills 的 SKILL.md 格式是跨平台开放标准。你在 Codex 里的 Skills 可以直接复制到 WorkBuddy，同一套 SKILL.md 在两边都能激活。

操作：cp -r ~/.codex/skills/某个Skill ~/.workbuddy/skills/某个Skill
例如你在 Codex 里装了 Humanizer-zh，复制后在 WorkBuddy 里也能用："用 humanizer-zh 润色这段文字"。

为什么这么做：Codex 适合长流程创作，WorkBuddy 适合碎片化日常任务——同一套 Skill，选对的工具跑对的任务。
🤖 每日一句：写 Skill 时把自己当作"流程教练"
好的 Skill 不是给你一套"正确答案模板"，而是给你一个你可以审阅的流程。codex-ppt-skill 的作者说："Skills 本质上是高度个性化的流程"。装完后别照搬——按你自己的偏好改大纲模板、固定风格、指定输出目录。让 Skill 越来越像"你自己做这件事的方式"。
