📗待学习：NotebookLM & Notion 每日进步
🧪 NotebookLM — 技巧 1：用"音频摘要"把文档变成播客
操作步骤：
1. 在 NotebookLM 中打开任意笔记本，点击右上角 "生成" 按钮
2. 选择 "音频摘要"（英文界面显示为 "Audio Overview"）
3. 等待 2-3 分钟，系统会生成一段两人对话式的音频（像播客一样）
4. 点击播放，可以边听边做其他事

适合场景：长文档/论文/报告，没时间全文阅读时，用音频快速掌握核心观点。支持中文文档，但生成的音频是英文（2026年6月状态）。
🧪 NotebookLM — 技巧 2："对话模式"深度挖掘文档细节
操作步骤：
1. 在笔记本里选择 "对话" 模式（不是"笔记"模式）
2. 用自然语言提问，NotebookLM 会基于你上传的所有文档回答
3. 高级用法："对比文档 A 和文档 B 中关于 XX 观点的差异" → 它会跨文档分析

今天就能试的提问模板：
• "这份报告的核心结论是什么？用 3 句话说清楚。"
• "文档里提到的 XX 方法，具体步骤是什么？"
• "把所有提到'风险'的地方找出来，列出原文。"

💡 提示：NotebookLM 的回答会附带来源引用（显示"来自：文档 X 第 Y 页"），可以点击跳转到原文验证。
🧪 NotebookLM — 技巧 3：把 NotebookLM 的笔记导入 Google Docs
操作步骤：
1. 在 NotebookLM 对话界面，找到 AI 生成的回答/笔记
2. 点击回答下方的 "导出到 Google Docs" 按钮
3. 系统会自动创建一个 Google Docs 文档，内容包含对话历史和引用来源
4. 可以在 Google Docs 里继续编辑、分享、协作

效率提升：研究 → 整理 → 协作，全流程在 Google 生态里完成，不用手动复制粘贴。
📋 Notion — 今日技巧：用 AI 快速生成数据库视图
操作步骤：
1. 在 Notion 数据库页面，点击 "添加视图"
2. 在视图类型选择界面，找到 "AI 生成" 选项（Notion AI 功能）

📙实用技巧：WorkBuddy 与 Agent
⚡ 今日技巧：把"常用提示词"存成快捷指令（Skill）
场景：你经常让 AI 帮你做某类固定任务（比如"帮我总结这篇文章"、"帮我写周报"），每次都要重新描述需求，很麻烦。

解决：把这类任务写成 WorkBuddy Skill，以后一键调用。

今天就能做的完整操作步骤：
  1. 在 WorkBuddy 里，说出你的需求（比如"帮我写一个'周报生成'的快捷指令，输入本周完成的工作列表，输出格式化的周报"）
  2. WorkBuddy 会帮你生成一个 SKILL.md 文件，保存到 ~/.workbuddy/skills/ 目录
  3. 以后直接用："/周报生成 [粘贴本周工作列表]" → WorkBuddy 自动按预设格式输出

📝 示例 SKILL.md（周报生成）：
--- name: weekly-report description: 根据用户输入的本周工作列表，生成格式化的周报。触发词：周报、weekly report --- # 周报生成 Skill ## 任务 将用户输入的工作列表整理为结构化周报。 ## 输出格式 **本周完成：** - [工作项 1：结果/产出] - [工作项 2：结果/产出] **下周计划：** - [计划项 1] - [计划项 2] **风险与阻碍：** - [如有，列出]

💡 进阶：去 obra/superpowers 参考它的 SKILL.md 写法——description 字段要写成"路由规则"（Claude 靠它判断是否激活），而不是营销文案。
🤖 Agent 小技巧：让 Agent 自己判断何时激活 Skill
关键点：Skill 的 description 字段决定了它何时被激活。写得好的 description 读起来像路由规则，不是宣传语。

好的 description 示例：
"Use when user needs to interact with PDF files: extract text, fill forms, extract tables. Triggered by keywords: PDF, 提取, 表单, 表格"

差的 description 示例：
"A powerful AI-powered skill that supercharges your workflow..."（Claude 看不懂什么时候该激活）

今天就能做的：检查你已安装的 Skills 的 description 字段，把"营销文案"改成"路由规则"。激活率能从 20% 提升到 90%。
