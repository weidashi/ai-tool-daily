changelog-generator — 产品更新日志自动生成
ComposioHQ · Git 提交 → 用户友好的更新日志
内容创作自动化产品运营
是什么：扫描 Git 提交历史，把技术 commit 自动翻译成用户看得懂的更新日志。按功能新增、改进优化、Bug 修复、安全更新自动分类，过滤掉内部重构和测试代码（用户不关心这些），输出干净的 changelog。支持指定版本范围、自定义分类规则、套用品牌语气。
最适合：产品经理、运营、开发者——每周五或发版日，一键生成产品更新日志，不用手动翻 commit 记录。

🎯 今天就能试：在你的项目目录里说「Create a changelog for commits from the past 7 days」，看它怎么把一堆技术 commit 变成用户友好的更新说明。

输入："Create a changelog for commits since v2.4.0, 按功能/优化/修复分类，语气友好"

NotebookLM 每日进步
🧪 闪卡与测验——从阅读工具升级为学习助理
上传课程资料后，点击「学习指南」中的功能：测验自动出选择题并附答案解析，适合自我检测；闪卡制作抽认卡（正面问题、背面答案），适合背名词解释。技巧：只想复习特定章节，只勾选部分来源资料再生成测验题。这让 NotebookLM 从「读文件工具」升级成「学习助理」。
🧪 协作分享笔记本——团队共享研究成果
点右上角分享按钮，生成只读链接发给同事。对方不需要重新上传文件，直接基于你整理好的源问问题。团队共享研究成果效率拉满——一个人整理，全团队受益。如果需要共同编辑，也可以设置编辑权限，多人往同一个笔记本添加素材。
🧪 Perplexity + NotebookLM 组合拳
外部探勘交 Perplexity（涵盖更广的 web 搜索）→ 把报告导入 NotebookLM 做交叉比对。Perplexity 负责「找」，NotebookLM 负责「整」。比单独用任一个都强——Perplexity 找得快但容易散，NotebookLM 整理得好但来源有限。两者组合是当前最强的 AI 研究工作流。
