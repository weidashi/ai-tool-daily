周日适合整理工具。今天不追热点，而是介绍 4 个真正能改变你日常内容生产方式的 Skill——从 Markdown 一键出公众号 HTML（样式不丢）、到全流程内容工厂（热点追踪到草稿箱）、到把 PDF 书变成 AI 可随时查阅的知识库、再到让 AI 自动匹配审美来排版——装好这四个，周一开工就有新武器。

Codex 日常效率 Skills
①
gzh-design-skill
isjiamu · Markdown→公众号HTML，全内联样式，粘贴不掉格式
⭐ 1.7K公众号排版Codex 原生新发布
是什么：专门为公众号排版设计的 Codex Skill。你把 Markdown 写完，它按你选的主题生成样式全内联的 HTML——粘贴到公众号编辑器后格式不丢、样式不掉。自动编章节号、标关键词下划线、配引言卡与目录、处理代码块和图片、合并作者签名，并用校验脚本兜住公众号平台的各种坑。
最适合：每次发公众号文章都要手动调格式的人——尤其是写长文、干货文的技术号/知识号作者。
📦 安装
git clone https://github.com/isjiamu/gzh-design-skill.git ~/.codex/skills/gzh-design-skill
安装后重启 Codex，对 Codex 说「用 gzh-design-skill 把这篇 Markdown 转成公众号 HTML」即可触发。
🎯 今天就能试：把你最近写好但还没发的那篇 Markdown 文章给它。选一个你喜欢的主题（比如"极简杂志风"或"温暖阅读风"），它会生成全内联 HTML——复制粘贴到公众号后台，格式完美保留，一个字都不用调。
📌 实操示例
输入："用 gzh-design-skill 把这篇关于 AI 写作的 Markdown 转成公众号 HTML，选「极简杂志风」主题，正文小标题要有编号。"
成果：一篇结构清爽的公众号 HTML。H2 标题自动编号（一、二、三），关键词有下划线强调，引言自动生成卡片样式，代码块带浅灰背景。复制到公众号后台，所有样式完好——不需要再手动调字号、行距、颜色。

WeWrite
imraywang · 公众号全流程工厂 v3.3——从热点抓取到微信草稿箱
⭐ 热门全流程最新 v3.3Codex/Claude 通用
是什么：一个把公众号内容生产拆成 7 个可独立调用节点的 Skill：选题 → 写作 → 质量评分 → AI 配图 → 16 主题排版 → 草稿箱推送 → 多平台改写。v3.3 刚于 7 月 13 日更新，退役了单体构建，改为 folder-per-skill 原生支持，三大宿主（Codex/Claude Code/OpenClaw）都能直接用。每个环节可以单独触发，也可以一句话跑完全程。
最适合：写公众号需要全流程自动化的人——尤其是有固定发文节奏、每次要从零搭建的创作者。
📦 安装
git clone https://github.com/imraywang/wewrite.git ~/.codex/skills/wewrite
v3.3 要求 folder-per-skill 结构，每个 Skill 子目录自带 SKILL.md。安装后重启 Codex。
🎯 今天就能试：对 Codex 说「用 wewrite 帮我想 3 个公众号选题，方向是 AI 工具推荐，目标读者是 25-35 岁职场人」。它会调用选题节点，输出 3 个带标题、hook、大纲的完整选题方案。
📌 实操示例
输入："用 wewrite 选题节点：帮我找 3 个「个人知识管理」方向的公众号选题，要求有信息差、能做成2000字干货文。"
成果：3 个选题方案，每个含：选题角度、目标读者画像、标题 3 个版本、hook 策略、大纲（3-4 个子标题）、信息差来源建议。选一个满意的，下一句就能触发写作节点开始写正文。
③
book-to-skill
virgiliojr94 · 把 PDF 书籍一键转成 Codex Skill，让 AI「读完」整本书
⭐ 8.6K知识管理阅读效率AI 外脑
是什么：把技术书籍、行业报告、研究方法论 PDF 一键转为 Codex Skill。装好后，Codex 相当于「读完了」整本书——你可以在任何对话中随时查阅书中概念、引用原文、对比观点，不用重新翻书。对知识工作者来说，这相当于把读过的重要书籍变成了 AI 的长期记忆。
最适合：经常需要引用书籍内容的人——写文章查方法论、做研究翻参考书、日常工作需要随时调取某本书的核心框架。
📦 安装
npx skills add virgiliojr94/book-to-skill --agent codex --global
装好后，对 Codex 说「帮我把这本 PDF 转成 Skill」，它自动处理并安装到 ~/.codex/skills/。
🎯 今天就能试：找一本你最近在读但总是翻不完的书（PDF 格式），让 book-to-skill 把它转成 Skill。之后写文章需要引用时，直接问 Codex「这本书里怎么讲 XX 概念的？原文在哪一章？」，它秒回——不用再从书架翻出来。
📌 实操示例
输入："帮我把《写作这回事》这本 PDF 转成 Codex Skill，保留章节结构，让 AI 能随时查阅核心观点和原文。"
成果：一个安装在 Codex 中的 Skill。之后在任何对话中问「斯蒂芬·金在《写作这回事》里怎么讲「副词是毒药」这个观点的？」Codex 直接输出原文章节+原文引用+你自己的理解。一本书从书架吃灰变成 AI 随时能查的知识库。

最适合：不想花时间研究排版审美、需要「给了想法就出成品」的人。
📦 安装
git clone https://github.com/LIyvqi/chenly-wechat-studio.git ~/.codex/skills/chenly-wechat-studio
安装后重启 Codex。对 Codex 说「用 chenly-wechat-studio 把这篇做成公众号草稿，自动选择审美」。
🎯 今天就能试：把你今天的一点灵感——哪怕只是几行想法——告诉 Codex：「用 chenly-wechat-studio 帮我写一篇关于「AI 时代的笔记方法」的公众号文章，受众是职场人，希望读完能有行动冲动」。它自动完成选题判断→正文写作→审美匹配→排版→草稿 payload。
📌 实操示例
输入："用 chenly-wechat-studio 把这篇草稿做成公众号文章。自动选审美。我想传达的是「用 AI 但不依赖 AI」的态度，偏理性但不冷。"
成果：AI 自动判断为「深度观点文」类型，匹配简洁克制风格——黑灰色调+大量留白+单栏阅读流。生成的 HTML 可直接粘贴到公众号后台，审美匹配到位，不需要手动微调。

Codex 官方动态 & 使用技巧
⚡ ChatGPT 全局搜索上线（7/14）
ChatGPT 网页端、iOS 和 Android 同步上线跨聊天记录、项目、图片和文件的全局搜索。从侧边栏一个入口搜索所有内容，支持按类型筛选。以前某个对话里的关键信息，你可能要翻十几页——现在直接搜关键词秒定位。这对日常用 ChatGPT 做研究、写作的人来说是个大更新。
⚡ Custom Instructions 扩至 5000 字符（7/15）
付费用户的自定义指令上限从 1500 字符暴增到 5000 字符。这意味着你可以写更详细的风格指南、品牌规范、写作规则——相当于给 ChatGPT 设置了一份「员工手册」。如果你有固定的写作风格（语气、用词习惯、禁用表达），现在有足够空间一次性写完，不再需要每次对话重新交代。
⚡ Codex iOS 内联可视化 + 任务控制优化（7/13）
Codex 在 iOS 端新增了任务内联可视化——在任务进行中就能看到图表、图片等可视化内容，不用等任务结束。同时改进了任务创建和管理的体验，包括更可靠的新建任务链接、工具活动进度指示器。移动端用 Codex 做内容工作的体验大幅提升。
💡 本周使用技巧：把 Custom Instructions 写成「写作风格卡」
现在有 5000 字符空间，建议这样写：① 你的身份和读者画像（2-3 句）② 你的写作风格关键词（如"冷静分析型""朋友聊天型"）③ 禁用表达清单（如"当今时代""总而言之""毋庸置疑"）④ 希望 AI 模仿的典型句式（3-5 句你的原话）⑤ 格式偏好（字数、段落结构、标题风格）。写完后丢一段自己的旧文章进去测试，看 AI 是否符合风格，微调 2-3 轮就能稳定。

NotebookLM 每日进步
🧪 一个笔记本，四种输出——别再只用默认模式
NotebookLM 的 Audio Overview 有四种格式可切换：Deep Dive（默认双人深度对话）、The Brief（单人 2 分钟精要）、The Critique（双人批判性审阅）、The Debate（双人多视角辩论）。同一组源材料，切四种格式出四种完全不同的产物。长报告用 Deep Dive，会前速览用 The Brief，自查草稿用 The Critique，发现观点盲区用 The Debate。大多数人只用默认 Deep Dive，错过了另外 75% 的价值。
🧪 把 Briefing Doc 存回笔记本做新来源——递归提升答案质量
生成一份 Briefing Doc 后，把它作为新来源存回这个笔记本。之后 NotebookLM 会在自己的摘要和原始文档之间交叉推理，后续问题的答案质量明显提升。这相当于让 AI 先通读一遍做摘要，再在摘要基础上做深度分析——比直接对着原始文档提问靠谱得多。尤其是处理 20+ 份来源的复杂研究时，这一步能大幅减少胡编乱造。
🧪 Short Video Overview 全量推送：60 秒竖屏视频，80 种语言
NotebookLM 的 Short Video Overview 功能已全量推送——你的源材料一键生成60 秒竖屏短视频，附带同步幻灯片和 AI 旁白，支持 80 种语言输出。适合做社交媒体素材、内部培训短片、客户简报。把你的 PPT 源文件丢进去，5 分钟出一段讲解视频，不用学剪辑。

WorkBuddy 实用技巧
⚡ 四要素结构化指令法——一次对话出成品
WorkBuddy 效率天花板不在于「功能多」，而在于「指令写得有多准」。记住这个公式：目标 + 输入 + 格式 + 约束 = 完美指令。

❌ 差的：「帮我做个报告」
✅ 好的：「读取桌面 Q3 销售数据文件夹里所有 Excel，按区域汇总销售额和增长率，生成柱状图+饼图，输出一份 5 页 PPT，使用蓝色商务风格」

四个要素一个不缺时，WorkBuddy 基本不返工，一次出成品。用熟练后你会发现——之前觉得 AI「不够聪明」，其实是指令没说清楚。
⚡ 三层记忆系统——让 WorkBuddy 越用越懂你
WorkBuddy 有三层记忆：云记忆（自动学习你的偏好）、用户级记忆（你明确告诉它「记住」的事）、工作区记忆（每个项目的特定规则）。用好它的关键是重要偏好主动写入——对 WorkBuddy 说「以后生成的所有报告都用蓝色商务风格，字号不小于 16pt」，它会把这条规则记到用户级记忆里。下次再让它出报告，自动套用，不需要重复交代。

今日成果展示
🎯 一本书 → 一个 AI 知识库：把《写作这回事》装进 Codex
用 book-to-skill + WeWrite + gzh-design-skill 三个工具，完成「读一本书→写一篇文章→排版发布」全流程
场景：你最近读了斯蒂芬·金的《写作这回事》，想写一篇公众号文章分享书中最颠覆你认知的 5 个写作原则。

Step 1 — 把书变成 Skill：对 Codex 说「用 book-to-skill 把《写作这回事》PDF 转成 Skill」，30 秒后这本书变成了 Codex 可随时查阅的知识库。

Step 2 — 用 WeWrite 出选题和初稿：「用 wewrite 帮我写一篇公众号文章：主题是「斯蒂芬·金教我的 5 个反直觉写作原则」，目标读者是想提升写作能力的职场人，要求有书中原文引用、有我的个人实践体会。」WeWrite 自动调取刚装好的书籍 Skill，在写作中自然引用原文。5 分钟出完整初稿。

Step 3 — gzh-design-skill 排版：「用 gzh-design-skill 把这篇文章转成公众号 HTML，选极简杂志风。」30 秒出排版，复制粘贴到公众号后台即可发布。

总耗时：约 15 分钟（以前写这种带引用+个人体验的深度文章至少要 2-3 小时）。关键不是「更快」，而是引用更准确、结构更清晰、排版不用手动调——省下的是最耗精力的那部分。
🌱 好的工具不只是让你更快，而是让你有精力去做只有你才能做的事——比如那个只有你经历过的故事、那个只有你思考过的观点。
