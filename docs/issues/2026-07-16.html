从"用别人的Skill"到"创建自己的Skill"

skill-creator · 咨询级PPT · Obsidian知识库 · 素材自动入库
📌 本期导读
ChatGPT Work 上线一周了。Codex 并入桌面端、Scheduled Tasks 定时跑、Record & Replay 录操作变 Skill——这些新能力最大的受益者不是程序员，而是每天跟文档、PPT、笔记打交道的知识工作者。但关键是：你怎么把「别人的 Skill」和「自己的需求」对接起来？本期从 skill-creator 讲起，四个 Skill 覆盖「创建→产出→管理→入库」全链路，最后教你用 skill-creator 给自己做一个专属的公众号选题诊断 Skill。

Codex 日常效率 Skills
①
skill-creator
vercel-labs · AI 技能的 npm：一句话创建你自己的专属 Skill
⭐ 24K+（仓库）Skill创建零代码68+ Agent兼容
是什么：Vercel 推出的 Skills CLI 生态中的核心能力。你对 Codex 说「帮我创建一个 XX 的 Skill」，skill-creator 会自动询问你的需求、触发场景、输出格式，然后生成一份完整的 SKILL.md 文件。装一次，以后任何重复性工作都能固化成 Skill——就像给自己的 AI 写一份标准操作手册。
最适合：有固定工作流程但不想学编程的知识工作者。写公众号、做周报、整理会议纪要、管理素材——任何你每周重复做的事，都能变成一个 Skill。
📦 安装
npx skills add vercel-labs/agent-skills --skill skill-creator --agent codex --global
装好后，Codex 中直接说「用 skill-creator 帮我创建一个 XX Skill」即可触发。
🎯 今天就能试：对 Codex 说「用 skill-creator 帮我创建一个『公众号选题诊断』Skill。我要它帮我分析一个选题值不值得写，给我3个标题方案和2个hook方向。触发条件是我说『帮我诊断这个选题』。」Codex 会自动生成一份符合你个人风格的 SKILL.md，以后每次只需要一句话就能跑完整套选题诊断流程。
📌 实操示例
输入：用 skill-creator 创建一个「周报自动生成」Skill：每天下午5点触发，读取我今天的 Codex 会话记录，抽取工作要点，按三段式（本周进展→问题与风险→下周计划）生成周报草稿，发到桌面。
成果：一份完整的 SKILL.md，定义了触发词、输入要求、三步工作流、输出格式。以后每天说「出今日周报草稿」→ Codex 自动按你定好的流程执行。不用每天重新教 AI 怎么做。

CyberPPT
crazyykhllc-bit · 把 DOCX/PDF/研究报告变成咨询公司级可编辑 PPTX
⭐ 743PPT生成咨询级8种风格可编辑
是什么：一个不走「快速出片」路线的 Codex Skill。给你一份文档或研究报告，它模仿麦肯锡的 PPT 品控流程——先建立证据表、脑暴故事线、SCR（现状-矛盾-方案）框架收敛，再生成可编辑的 PPTX。16 道质量门禁、8 种视觉风格（深红咨询风、象牙白金融风、冷灰科技风等），每页关键文字和数字保持可编辑。
最适合：咨询顾问做客户提案、项目经理做复盘汇报、研究人员把报告转成演示文稿。尤其适合「文档已经写好了，但要变成能上台讲的 PPT」这个场景。
📦 安装
git clone https://github.com/crazyykhllc-bit/CyberPPT.git ~/.codex/skills/cyber-ppt
装好后在 Codex 中说「帮我把这份 PDF 转成咨询风格 PPTX，目标 25 页，用风格 04 象牙白+深蓝强调」即可启动三阶段硬流程。
🎯 今天就能试：找一份你最熟悉的 PDF 报告（年度总结、行业分析、项目方案都行），丢给 Codex 说「用 CyberPPT 帮我把这份报告转成 PPT」。它会先分析内容、出故事线和大纲——你确认后再进入蓝图和生成阶段。三个确认门保证不跑偏。
📌 实操示例
输入：帮我把「2026Q1 行业研究报告.pdf」转成咨询风格 PPTX，目标页数 25 页，使用风格 01 经典深红咨询风。受众是公司高管，需要突出关键数据和战略建议。
成果：一份 25 页 .pptx 文件——封面+目录+Executive Summary+逐章分析+数据图表+结论建议。主标题、正文、关键数字、页码全部可编辑。视觉效果接近麦肯锡/BCG 风格的正式交付物。
③
obsidian-skills
kepano（Obsidian CEO） · 教会 AI 说 Obsidian 的「母语」
⭐ 35.9K知识管理ObsidianPKM零配置
是什么：Obsidian CEO Steph Ango 亲自写的 5 个 Skill 文件。装上后，Codex 自动学会 Obsidian 的所有原生语法——[[双向链接]]、callout 标注框、YAML 元数据、Canvas 画布、Bases 结构化数据库。之前让 AI 写 Obsidian 笔记总差那么一点（链接格式不对、不会用 callout），现在完全原生。
最适合：用 Obsidian 做个人知识库（PKM）的知识工作者。让 Codex 帮你创建结构化的笔记、自动建立笔记之间的链接、生成 Canvas 视觉图谱——AI 真正理解你的笔记系统。
📦 安装
git clone https://github.com/kepano/obsidian-skills.git ~/.codex/skills/obsidian-skills
5 个子 Skill 自动识别：obsidian-markdown（语法）、obsidian-bases（数据库）、json-canvas（画布）、obsidian-cli（命令行）、defuddle（网页提取）。
🎯 今天就能试：对 Codex 说「帮我在 Obsidian 里创建一篇关于『AI 工具选型方法论』的笔记，用 callout 标注关键原则，列出 5 个选型维度，每个维度链接到对应的工具笔记，并在 YAML 里标记 tags 和创建日期。」Codex 生成的笔记会完美符合 Obsidian 格式，可以直接粘贴使用。
📌 实操示例
输入：帮我分析这3篇关于知识管理的文章（粘贴链接），在 Obsidian 中创建一篇综合笔记。用 callout 标注三篇文章的核心观点异同，用 [[双向链接]] 关联到已有的「PKM系统设计」笔记，生成一个 Canvas 画布展示三者关系。
成果：一篇格式完美的 Obsidian 笔记——标准 YAML frontmatter + [[双向链接]] + callout 对比框 + Canvas 关系图。省去了手动调整链接格式和 callout 语法的所有时间。

Obsidian Inbox
Simuiu · 视频/文章/网页/微信内容 → 结构化 Obsidian 笔记 + 自动索引
⭐ 新发布素材入库自动索引Local-First
是什么：一个 local-first 的知识入库 Skill。把外部视频链接、文章、网页、微信内容甚至本地媒体文件丢给它，自动转成结构化 Obsidian Markdown 笔记，并维护三个可点击的索引：资料库索引、主题索引、用途索引。不是一次性摘要——每条资源都有原始材料留存、转写文本归档、普通总结和深度总结分层。
最适合：每天在微信、B站、公众号里收藏大量内容但从来不知道放哪的人。帮你把「收藏即遗忘」变成「收藏即入库」。
📦 安装
git clone https://github.com/Simuiu/obsidian-inbox.git ~/.codex/skills/obsidian-inbox
装好后对 Codex 说「帮我把这个 B站视频/公众号文章 存入 Obsidian Inbox」即可触发自动入库流程。
🎯 今天就能试：找一个你今天读到的好文章（公众号文章链接或网页 URL），对 Codex 说「帮我把这篇文章用 Obsidian Inbox 存到我的知识库里，做普通总结就行」。它会自动抓取正文、生成结构化笔记、更新索引文件。
📌 实操示例
输入：帮我把这个 B站视频（链接）和这篇公众号文章（链接）用 Obsidian Inbox 存起来，视频做普通总结，文章做深度总结（因为这篇文章很重要）。
成果：两个资源的笔记 MD + 原始转写/正文归档 + 资料库索引更新 + 主题标签自动提取。以后想看这篇文章的核心观点，打开 Obsidian 搜一下就有，不用在微信收藏里翻半天。
skill-creator→ CyberPPT→ obsidian-skills→ Obsidian Inbox 创建专属工具 → 高质量产出 → 知识结构化 → 素材自动入库。四个 Skill 串成完整链路。

Codex 官方动态 & 使用技巧
🔥 ChatGPT Work + Scheduled Tasks：不写代码，也能定时让 AI 干活
ChatGPT Work 是 GPT-5.6 驱动的新代理模式——可以跨应用、跨文件、持续数小时完成任务。对非程序员来说，两个最实用的能力是：（1）Scheduled Tasks 定时任务——设置「每周一早上汇总上周产品更新」「每天检查共享表格是否有待办」「监控指定来源有新内容时生成摘要」，AI 按时自动执行；（2）@ 应用联动——在提示词中 @Slack、@Google Drive、@Gmail，ChatGPT Work 自动从对应平台拉取信息并处理。在 ChatGPT 桌面端的 Scheduled 面板创建任务，设定时间规则即可。
🎥 Record & Replay：录一次操作，AI 自动学会你的 SOP
Codex 的新功能（macOS，需 Computer Use）：打开 Plugins → Record a skill → 演示一遍你的操作（比如填报销单、发布视频到多平台、下载定期报告）→ 停止录制，Codex 自动生成一份 SKILL.md。下次说「用上次那个报销 Skill 帮我填这个月」，它就能重放整条流程。以前教 AI 靠写提示词，现在直接做一遍给它看——比 RPA 工具简单 10 倍。
📋 AGENTS.md：给 Codex 写一份「员工手册」
在 Codex 设置 → 个性化 → 自定义命令中创建 AGENTS.md，把「公众号排版规范」「PPT 配色要求」「邮件语气偏好」「禁用词列表」等固定规则写进去。之后每次让 Codex 生成内容，它自动按这些规则执行，不用每次都重新说一遍。分为全局级（所有项目通用）和项目级（单个项目专属）。这才是让 AI 真正「记住你的偏好」的正确方式。

NotebookLM 每日进步
📱 Video Overview：60 秒竖屏短视频摘要
NotebookLM 新增 Video Overview 功能。上传资料后，NotebookLM 可以生成 TikTok 风格的 60 秒竖屏短视频摘要——有配乐、文字滚动、核心观点高亮。三种格式可选：Explainer（多语言支持）、Cinematic（电影感）、Short（60秒快剪，需 Google Pro/Ultra）。把研究报告丢进去，通勤路上刷一遍视频就消化完了。
🧪 反向解释检验法：检验理解的最快方式
对 NotebookLM 说：「用错误的方式向我解释这个概念。常见的误解是什么？」——知道一个想法的「错误版本」，往往比知道正确版本更能让你真正理解它。你读到误解时心里会想「哦，我明白为什么会有人这么想了，但实际上——」，然后对正确答案的理解就深了一层。
🎬 Studio 自定义指令：做你自己的 AI 播客制作人
在 Studio 面板的 Audio Overview 中，可以自定义两位 AI 主持人的讨论重点、受众背景和对话风格。比如设置「目标受众是 25-35 岁的职场人，用通俗语言解释专业概念，每次讨论结尾给出 3 条实操建议」，生成的播客就完全按你的设定走。四种对话格式可选：Brief（简报）、Critique（批判）、Debate（辩论）、Deep Dive（深度探讨）。

今日成果展示
🎯 用 skill-creator 给自己做一个「公众号选题诊断」Skill
不用写一行代码，和 Codex 对话就能创建。以后每次只需要一句话跑完整套流程。
步骤：

1. 安装 skill-creator：
npx skills add vercel-labs/agent-skills --skill skill-creator --agent codex --global

2. 对 Codex 说：
「用 skill-creator 帮我创建一个『公众号选题诊断』Skill。能力要求：
• 触发词：我说「帮我诊断这个选题」
• 分析维度：受众明确性、痛点清晰度、信息差大小、传播潜力评分（1-10）
• 输出：诊断结论 + 3 个标题方案 + 2 个 hook 方向 + 建议的写作框架
• 风格：直接、不留情面，不好的选题直接说不好」

3. 成果：
Codex 自动生成一份完整的 SKILL.md，保存在 ~/.codex/skills/ 目录下。

4. 以后这样用：
对 Codex 说：「帮我诊断这个选题：『AI 时代，知识管理还需要记笔记吗？』目标读者是 25-35 岁的职场知识工作者。」

📦 你会得到：
【诊断结论】选题成立，受众明确（职场人），痛点是真实存在的竞争性需求（记笔记 vs AI 搜索）。信息差中等——建议从「笔记的心理价值 vs 检索效率」的角度切入，不要停留在「该不该记」的二元讨论。
【传播潜力】8/10——话题有天然争议性，容易引发讨论和转发。
【3 个标题方案】① AI 时代你还在记笔记？资深笔记党告诉你：记和不记，差的不是信息量 ② 我让 AI 帮我读了三个月论文，然后发现了一件可怕的事 ③ 知识管理的终点不是「记住」，是「想通」
【2 个 hook 方向】① 反常识：研究显示手写笔记的记忆留存率比 AI 摘要高 47% ② 场景共鸣：描述一个「明明看过但又忘了」的尴尬瞬间
【建议写作框架】hook 入口 → 抛出核心矛盾 → 引用研究数据 → 个人实践体验 → 给出可操作结论 → 引发评论互动
